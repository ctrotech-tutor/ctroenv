# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in CtroEnv, please report it by
opening a [GitHub Security Advisory](https://github.com/ctrotech-tutor/ctroenv/security/advisories/new).

Do NOT report security vulnerabilities through public GitHub issues.

## Response

We will acknowledge receipt of your report within 72 hours and provide an
estimated timeline for a fix.

## Scope

Security issues include, but are not limited to:

- Exposure of environment variable values in error messages
- Secret masking bypasses
- Code injection via schema definitions
- Dependency vulnerabilities
